字符串相关的问题在面试的时候碰到的概率极高，虽然平时工作中，因为有成熟的框架和各种库，我们很少碰到这样的需求，但是解决问题的思路是需要我们了解的，做一个优秀的程序员要了解一些基本的算法。

字符串翻转

一般来讲，写代码翻转一个C风格的字符串，那就是考察基本功了。但需要注意的是，能不能使用strlen()这个方法，使用和不使用，解法就不一样。

void swap(char &a, char &b) {
    char temp = a;
    a = b;
    b = temp;
}

void reverseStringWithoutStrlen(char *str) {
    //
    if (NULL == str) {
        return;
    }
    //
    char *beg = str;
    char *end = str;
    while (NULL != *end) {
        end++;
    }
    end--;
    //
    while (beg < end) {
        swap(*beg, *end);
        beg++;
        end--;
    }
}

void reverseStringWithStrlen(char *str) {
    if (NULL == str) {
        return;
    }
    int len = strlen(str);
    for (int i = 0; i < len / 2; i++) {
        swap(str[i], str[len - i - 1]);
    }
}

旋转字符串

这个题目和上一个不同，这个需求是这样的：给定一个字符串，要求把字符串前面的若干个字符移动到字符串的尾部，如把字符串“abcdef”前面的2个字符’a’和’b’移动到字符串的尾部，使得原字符串变成字符串“cdefab”。请写一个函数完成此功能，要求对长度为n的字符串操作的时间复杂度为 O(n)，空间复杂度为 O(1)。最直观的解决办法就是暴力移位，先把a移到最后，然后再移b，这样就能完成任务，但是时间复杂度就是O(m*n)了，空间复杂度是O(1)，明显达不到性能要求。目前比较好的解决办法，是3步反转法，解决思路是这样的，例如字符串abcdef长度是6，需要逆序2个字符ab，那么用以下办法 1.逆序整个字符串得到fedcba 2.将字符串前面4个字符翻转一次cdefba 3.将字符串后面剩下的2个字符逆序cdefab 下面是代码

void reverse(char *left, char *right) {
    while (left < right) {
        char temp = *left;
        *left = *right;
        *right = temp;
        left++;
        right--;
    }
}

void reverseString(char *str, int k) {
    //
    if (NULL == str) {
        return;
    }
    //
    int len = strlen(str);
    reverse(str, str + len - 1);
    reverse(str, str + len - k - 1);
    reverse(str + len - k, str + len - 1);
}

字符串转为整数

题目的要求是输入一个由数字组成的字符串，把它转换成整数并输出。例如：输入字符串”123”，输出整数123，不能使用库函数atoi。这个问题直接想，觉得还挺简单的，但你写一下，就会发现没有思虑清楚。一般解决这种问题的思路是这样的：1、首先判断字符串是否为空，返回0。2、考虑前面的是否是空格，如果是的话，要去掉。3、判断第一个字符是不是+和-，设置变量符号。4、循环取得字符串的数字，考虑字符串中有非数字，遇到就退出，保留前面的数字。5、要考虑溢出的情况，溢出返回Integer的最大值或最小值。下面是代码

int StringToInt(char *str) {
    const int INT_MAX = 2147483647;
    const int INT_MIN = -2147483648;
    long long sum = 0;
    //
    if (NULL == str) {
        return 0;
    }
    //
    while (' ' == *str || '\t' == *str)str++;
    //
    int sign = 1;
    if ('-' == *str) {
        sign = -1;
        str++;
    } else if ('+' == *str) {
        sign = 1;
        str++;
    }
    //
    while (*str >= '0' && *str <= '9') {
        //
        int c = *str - '0';
        //
        if (sign > 0) {
            if (sum > INT_MAX / 10 || (sum == INT_MAX / 10 && c > INT_MAX % 10)) {
                sum = INT_MAX;
                break;
            }
        } else {
            if (sum > (unsigned) INT_MIN / 10 || (sum == (unsigned) INT_MIN / 10 && c > (unsigned) INT_MIN % 10)) {
                sum = INT_MIN;
                break;
            }
        }
        //
        sum = sum * 10 + c;
        str++;
    }
    //
    return sum;
}

字符串包含

给定一长一短的两个字符串A和B，假设A长B短，要求判断B中的所有字母是否都在字符串A中。解决办法有几种，第一个就是暴力的，B中的每个字母都去A中遍历，那么时间复杂度就是O(n*m)。再优化一下，如果可以的话，先把A和B都排序了，然后再同时对两个字串依次轮询，两个字串的排序需要O(mlogm) + O(nlogn)次操作，之后的线性扫描需要O(m+n)次操作。那么我们再考虑一下，有没有可能再快点？假设有一个仅由字母组成字串，让每个字母与一个素数对应，从2开始，往后类推，a对应2，b对应3，c对应5，……。遍历第一个字串，把每个字母对应素数相乘。最终会得到一个整数。利用上面字母和素数的对应关系，对应第二个字符串中的字母，然后轮询，用每个字母对应的素数除前面得到的整数。如果结果有余数，说明B中有字母不在A中。如果整个过程中没有余数，则说明第二个字符串是第一个的子集了。这个方法有一个缺点，缺点是素数相乘的结果容易导致整数溢出，处理比较长的字符串有问题。更好的办法就是，可以先把长字符串a中的所有字符都放入一个Hashtable里，然后轮询短字符串b，看短字符串b的每个字符是否都在Hashtable里，如果都存在，说明长字符串a包含短字符串b，否则，说明不包含。

bool StringContain(string &a, string &b) {
    int hash = 0;
    for (int i = 0; i < a.length(); ++i) {
        hash |= (1 << (a[i] - 'A'));
    }
    for (int i = 0; i < b.length(); ++i) {
        if ((hash & (1 << (b[i] - 'A'))) == 0) {
            return false;
        }
    }
    return true;
}
