ubuntu域名暂时不能解析问题

我们新安装的ubuntu机器，在sudo apt-get update的时候，有时会出现 Temporary failure resolving 'us.archive.ubuntu.com'错误，根本原因还是域名解析出了问题。有的小伙伴直接修改/etc/resolv.conf，添加nameserver来解决问题，但这是不足取的。因为/etc/resolv.conf文件头部已经提示你了

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN

让你不要直接修改这个文件。最好的办法还是在/etc/resolvconf/resolv.conf.d/base文件里添加如下内容

nameserver 223.5.5.5
nameserver 223.6.6.6

然后执行sudo resolvconf -u更新resolvconf，然后我们看一下cat /etc/resolv.conf，如果有如下内容

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 223.5.5.5
nameserver 223.6.6.6

说明就设置成功了，再sudo apt-get update试一下，问题应该解决了。
