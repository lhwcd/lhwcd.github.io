Docker创建持久化Redis容器

By lhw

2018年07月20日

文章目录

我想要的功能是输入经纬度，能够查得到省市县的地名。同时还要查询速度快，能够独立部署。所以就不能使用在线查询，因为网络情况不可靠，而且也太慢了。所以我考虑使用redis的geo功能来做这个工作。选择使用docker安装redis，而不是原生安装，是因为不想碰那些配置和环境变量这些，同时方便以后项目的迁移。下面就是我的安装步骤， 首先要确定到底安装哪个版本的redis镜像，我们要通过命令docker search redis在仓库里找一下

lhw@worker:~$ docker search redis
NAME                                 DESCRIPTION                                     STARS               OFFICIAL            AUTOMATED
redis                                Redis is an open source key-value store that…   5475                [OK]                
bitnami/redis                        Bitnami Redis Docker Image                      82                                      [OK]
sameersbn/redis                                                                      68                                      [OK]
tenstartups/redis-commander                                                          32                                      [OK]

我们找到了OFFICIAL的redis，我决定使用这个官方版本，于是使用命令docker pull redis把这个镜像拉下来

现在我们再看一下我们已经拥有的image

lhw@worker:~$ docker image ls
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
redis               latest              f06a5773f01e        2 days ago          83.4MB
hello-world         latest              e38bc07ac18e        3 months ago        1.85kB

下面我们就启动redis容器

lhw@worker:~$ docker run -p 6379:6379 -v ～/data:/data -d redis:latest redis-server --appendonly yes
9f3ad3d6fd4969f3ee6d338659af9544773523e98240132d191b3e0fa011e7ed
lhw@worker:~$ docker container ls
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS                    NAMES
9f3ad3d6fd49        redis:latest        "docker-entrypoint.s…"   8 seconds ago       Up 6 seconds        0.0.0.0:6379->6379/tcp   vigilant_fermi

docker run命令用于运行一个新容器，-p用于映射宿主机指定端口到容器指定端口，-v命令会将宿主机的～/data挂载到容器中，并绕过联合文件系统，我们可以在主机上直接操作该目录。任何在该镜像/data路径的文件将会被复制到Volume。-d是detach的缩写，表示在后台运行且打印容器的ID，后面的--appendonly yes是传给redis-server的命令的，表示打开AOF持久化存储。 然后我就把准备好的命令文件导入到redis中

cat lnglat.txt | redis-cli

这样，在我用户主目录的/data目录下，就有一个appendonly.aof文件，以后这个文件就可以迁移了
