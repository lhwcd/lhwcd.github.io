链表是一个工作中常用的数据结构，在面试中也会经常碰到，这里记录一些常见的算法，希望对大家有帮助。本文对链表节点的定义如下

typedef struct Node {
    int data;
    Node* next;
};

逆转单链表

将一个单链表的逆序反转，有两种思路。第一种就是非递归，用pre、cur、next在链表上一个循环。第二种递归算法是先逆转下一个节点，再逆转当前节点，递归不好理解，也在C++下有调用递归深度的限制，不推荐。 非递归算法这里，我们使用了三个指针：pre、cur、next。其中cur其实可以使用head来代替，这样就会少用一个指针。但为了程序读起来更清楚，我还是坚持多用一个cur指针。

Node *reverseByLoop(Node *head) {
    //
    if (NULL == head || NULL == head->next) {
        return head;
    }
    //
    Node *pre = NULL;
    Node *cur = head;
    Node *next = NULL;
    while (cur != NULL) {
        next = cur->next;
        cur->next = pre;
        pre = cur;
        cur = next;
    }
    return pre;
}

单链表是否有环

现在有一个单向链表，判断链表是否有环存在？通过快慢指针来解决，通过两个指针，分别从链表的头节点出发，一个每次向后移动一步，另一个移动两步，两个指针移动速度不一样，如果存在环，那么两个指针一定会在环里相遇。需要注意的是，快慢指针在处理链表问题上经常碰到，碰到链表问题，想一想快慢指针是不是能解决问题。

bool checkCircle(Node *head) {
    //
    if (NULL == head) {
        return false;
    }
    //
    Node *fast = head;
    Node *slow = head;
    while (NULL != fast && NULL != fast->next) {
        fast = fast->next->next;
        slow = slow->next;
        if (fast == slow) {
            return true;
        }
    }
    return false;
}

单向有环链表中环的长度

这个问题其实比较好解决，因为通过快慢指针，能够找到重合的节点，记下重合点。然后再让一个指针转一圈，回到记下的重合点，通过记数就知道长度了。

int getCircleLength(Node *head) {
    if (NULL == head) {
        return 0;
    }
    Node *fast = head;
    Node *slow = head;
    while (NULL != fast && NULL != fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            break;
        }
    }
    if (fast != slow) {
        return 0;
    }
    int count = 1;
    while (slow != fast->next) {
        fast = fast->next;
        count++;
    }
    return count;
}

找到单向链表的环入口点

这道题相对不太直观，但还是可以理解的。我们由上题所知，fast每次2步，slow每次1步，最终他们在m点汇合了。接下来，我们从m点开始，fast每次走1步，而slow回到链表的头部，重新走，每次也是1步。那么当fast和slow相遇的时候，就是环路的入口了。问题来了，为什么是这样？假定起点到环入口点的距离为a，而相交点m与环入口点的距离为b，环路的周长的c，那么当slow和fast第一次相遇的时候，假定slow走的距离是d，那么我们知道：

a+b = d
a+b+kc = 2d

为什么a+b+kc = 2d？是因为fast每次走2步，slow是每次走1步，所以fast的速度是slow的2倍。由上面的两个公式，我们可以得到kc=d，也就是说，slow从链表头部走到相遇点m的距离，一定是环周长的整数倍，k是一个大于0的正整数。那么现在，如果fast从相遇点m开始，以每次1步在环内走，而slow从链表头部重新走，也是每次一步，他们一定会在m点再相遇的，这是由kc=d确定了的。而他们走过的路径，相同的部分是b，即相交点m与环入口点的距离，去掉这个b的共同路径，那么就是他们的环路的入口点了。所以当slow和fast第一次相遇的时候，一定就是环路的入口点。下面是代码

Node *GetCircleEntryNode(Node *head) {
    //
    if (NULL == head) {
        return NULL;
    }
    //
    Node *fast = head;
    Node *slow = head;
    while (NULL != fast && NULL != fast->next) {
        fast = fast->next->next;
        slow = slow->next;
        if (fast == slow) {
            break;
        }
    }
    if (fast != slow) {
        return NULL;
    }
    //
    slow = head;
    while (fast != slow) {
        slow = slow->next;
        fast = fast->next;
    }
    return fast;
}

找到链表的中间节点

找链表的中间节点，如果链表的长度为偶数，返回中间两个节点的任意一个，若为奇数，则返回中间节点。其实解决方案也可以通过快慢指针来完成，用两个指针从链表头节点开始，一个指针每次向后移动两步，一个每次移动一步，直到快指针移到到尾节点，那么慢指针即是所求。所以我们处理链表问题的时候，一定要用好快慢指针。下面是代码

Node *getMiddleNode(Node *head) {
    if (NULL == head) {
        return NULL;
    }
    Node *step1 = head;
    Node *step2 = head;
    while (NULL != step2 && NULL != step2->next) {
        step1 = step1->next;
        step2 = step2->next->next;
    }
    return step1;
}

求链表倒数第k个节点

输入一个单向链表，输出该链表中倒数第k个节点，链表的倒数第0个节点为链表的尾指针。解决思路类似快慢指针，但又稍有不同。设置两个指针slow和fast都指向head，然后fast向前走k步，这样slow和fast之间就间隔k个节点，最后slow和fast同时向前移动，直至fast走到链表末尾，那么slow指针就指向倒数第k个节点。下面是代码

Node *getKthNode(Node *head, int k) {
    //
    if (NULL == head || k <= 0) {
        return NULL;
    }
    //
    Node *slow = head;
    Node *fast = head;
    while (k-- >= 0) {
        if (NULL == fast) {
            return NULL;
        }
        fast = fast->next;
    }
    //
    while (NULL != fast->next) {
        fast = fast->next;
        slow = slow->next;
    }
    return slow;
}

逆序打印单链表

这个问题不难，主要有两个解决办法。第一个办法是使用stack，先压进去，出退出来，这样就逆序了。第二个办法就是使用递归，代码看起来比较简单。

void reversePrintList(Node *head) {
    //
    if (NULL == head) {
        return;
    }
    //
    reversePrintList(head->next);
    //
    std::cout << head->data << endl;
}

判断两个无环链表是否相交

解决问题的思路，第一个简单粗暴，直接循环判断第一个链表的每个节点是否在第二个链表中，这样明显时间复杂度是O(len(list1)*len(list2))。第二个解决思路，我们用空间换时间，直接把list1每个节点放到hash表中，然后将list2的每个节点在hash表中检查是否已经存在了，那么时间复杂度就变成了O(len(list1)+len(list2))，空间复杂度变成了O(len(list1))，也不是特别好。我们再转变一下思路，能不能把相交的问题，变成判断环的问题？把list2接在list1后面，再用前面判断有环的方法来确定是否有环，有环的话，说明有相交，效率也不是很好，因为时间复杂度也是线性的。那么我们还有没有其它办法？其实如果两个环相交，那么相交节点之后的所有节点都是两个链表共同所有的，所以我们只需要判断两个链表的尾指针是否相等就可以了。这个方法时间复杂度是O(len(list1)+len(list2))，空间复杂度是O(1)，下面是代码

bool checkListIntersect(Node *head1, Node *head2) {
    //
    if (NULL == head1 || NULL == head2) {
        return false;
    }
    //
    if (NULL != head1->next) {
        head1 = head1->next;
    }
    if (NULL != head2->next) {
        head2 = head2->next;
    }
    //
    return head1 == head2;
}

判断两个有环链表是否相交

如果有环且两个链表相交，则两个链表都有共同一个环，即环上的任意一个节点都存在于两个链表上。因此，就可以判断一链表上俩指针相遇的那个节点，在不在另一条链表上。


bool checkListIntersectWithLoop(Node *head1, Node *head2) {
    //
    if (NULL == head1 || NULL == head2) {
        return false;
    }
    //
    if (!checkCircle(head1) || !checkCircle(head2)) {
        return false;
    }
    //
    Node *node = findCircleNode(head1);
    if (NULL == node) {
        return false;
    }
    //
    while (NULL != head2) {
        if (node == head2) {
            return true;
        } else {
            head2 = head2->next;
        }
    }
    return false;
}

两个单链表的第一个公共节点

思路开始前，我们一定要想像出两个链表相交的样子，应该是一个Y字形的，这样就方便思考了。可以先遍历两个单链表得到长度，然后求得两个链表长度的差值，然后让长的那个链表先走到差值长度位置，然后两个链表再同时遍历，直到找到第一个公共节点。

Node *findFirstSameNode(Node *head1, Node *head2) {
    //
    int len1 = getListLength(head1);
    int len2 = getListLength(head2);
    //
    if (len1 > len2) {
        int diff = len1 - len2;
        while (diff-- > 0) {
            head1 = head1->next;
        }
    } else {
        int diff = len2 - len1;
        while (diff-- > 0) {
            head2 = head2->next;
        }
    }
    while (NULL != head1) {
        if (head1 == head2) {
            return head1;
        } else {
            head1 = head1->next;
            head2 = head2->next;
        }
    }
    return NULL;
}

链表问题总结

很多链表问题，为了提高效率，可以考虑多用几个指针，快慢指针就是一种常见的方法。所以碰到新的链表问题，就考虑多几个指针遍历，看看能不能解决问题，这个思想一定要有。
